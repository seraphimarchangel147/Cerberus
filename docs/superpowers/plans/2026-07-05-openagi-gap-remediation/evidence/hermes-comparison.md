Philosophy: hermes-agent is product-engineered maturity around a conventional agent loop — one AIAgent class reused across CLI/TUI/desktop/gateway/ACP, governed by two invariants (byte-stable prompt for cache preservation; a "narrow waist" ~45-tool core) with learning-as-files (skills, MEMORY.md, FTS5 recall) and strictly consent-first, conversation-derived proactivity. openAGI is a cognition-first research thesis — scrutiny→tiered-memory→propagation as a closed loop fed by an outcome store — bolted to a genuinely novel sensing layer hermes entirely lacks. In short: openAGI senses better; hermes acts, converses, and operates better.

openAGI's honest advantages: OS-level ambient capture is real and live (ScreenCaptureKit+Vision OCR, 742MB FTS5 observation store written daily; app-focus tracking; iMessage ingestion; calendar/Rize/BuildBetter transcript corroboration) — hermes has zero ambient sensing; its proactivity is conversation-derived only. openAGI enforces scrutiny verdicts as tool policy at invoke time (not prompt text), enforces per-specialist tool allowlists, feeds outcome quality into routing/retirement, self-tunes auto-complete thresholds from clarification history, has quiet-hours-aware digest outreach with native notification action buttons, a budget/credit ledger, corrections that lock in memory, and a main-and-nodes topology. Its Mac app (~3.5k lines Swift) is more polished than anything hermes ships for ambient UX.

But openAGI's execution layer is thin where hermes is deep. The agent host is 408 lines; the model provider hard-slices context to the last 12 messages (model-provider.js:88,237) with no summarization or cache economics, versus hermes's compression-with-session-split under a byte-stable cached prompt. openAGI's tool registry has ~35 domain tools (tasks, memory, cron, MCP, skills) but no terminal, no file tools, no browser, no web search — action happens only via MCP servers, skill replay, or off-by-default computer-use — versus hermes's terminal-on-6-backends, 12 browser tools, execute_code, and parallel delegation. Delivery: hermes runs ~28 messaging platforms with pairing security and per-user/thread session isolation; openAGI has Telegram/Twilio classes whose live store is empty — outreach is effectively Mac-notification-only. Operations: hermes has crash-recovery state machines, file-locked cron with catchup windows and hard interrupts, restart-loop suspension, and a hardened approval engine (YOLO frozen at import, dangerous-command patterns, persistent allowlists, constrained toolsets for untrusted webhook payloads); openAGI's cron has none of lock/catchup/timeout, and its pending-actions approval queue has never persisted an item.

The learning loops tell the same story: hermes's self-improvement (post-turn background review, /learn authoring, skill curator, bundled skill library, blueprint suggestions) is an exercised product path; openAGI's equivalents are starved — scrutiny fitter needs 50 outcomes and has 3, weights have never adapted, 74 mined skill candidates but 0 accepted skills, replay never run. Patterns worth adopting from hermes: cache-stable context with real compression, a general execution surface, channel breadth for outreach delivery, cron/ops hardening, on-demand cross-session transcript search, day-one skill library plus authoring, per-turn learning triggers, and injection-aware approval hardening — all grafted onto openAGI's superior sensing and outcome-feedback substrate.
